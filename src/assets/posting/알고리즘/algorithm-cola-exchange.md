## 문제 요약

프로그래머스 **콜라 문제(Lv.1)** 는 다음 규칙을 반복하는 문제다.

- 빈 병 `a`개를 가져가면 콜라 `b`병을 받는다.
- 현재 가진 빈 병 수가 `n`일 때, 최종적으로 받을 수 있는 콜라 병 수를 구한다.

핵심은 **교환을 몇 번 할 수 있는지**를 정확히 누적하는 것이다.

---

## 접근 1) 반복 시뮬레이션

가장 직관적인 방식은 교환 과정을 그대로 while문으로 돌리는 것이다.

```java
class Solution {
    public int solution(int a, int b, int n) {
        int answer = 0;

        while (n >= a) {
            int exchanged = (n / a) * b; // 새로 받은 콜라 수
            answer += exchanged;

            // 남은 빈 병 + 새로 받은 콜라를 마시고 생긴 빈 병
            n = (n % a) + exchanged;
        }

        return answer;
    }
}
```

### 왜 맞는가?

- `n / a`번 교환 가능
- 교환 후 빈 병은 `n % a`개 남음
- 받은 콜라를 마시면 빈 병 `exchanged`개가 다시 생김
- 따라서 다음 상태는 `(n % a) + exchanged`

### 복잡도

- 시간: 대략 `O(n)` (교환 횟수에 비례)
- 공간: `O(1)`

---

## 접근 2) 수학적 관찰

한 번 교환할 때마다 빈 병은 실질적으로 `a - b`개 감소한다.

- `a`개를 내고
- `b`개를 다시 받기 때문

그래서 반복문 없이 식으로도 풀 수 있다. 다만, **경계 조건(나누어떨어지지 않는 경우)** 을 매우 조심해야 한다.

실전 코딩테스트에서는 안정성을 위해 보통 시뮬레이션 풀이를 먼저 선택하는 편이 안전하다.

---

## 정리

- 이 문제는 시뮬레이션 대표 유형이다.
- 구현은 간단하지만 상태 전이(`n` 업데이트)를 틀리기 쉽다.
- 발표에서 비교한 것처럼, 시뮬레이션으로 먼저 맞춘 뒤 수학식으로 최적화 관점을 가져가면 좋다.
